# Six-feature expansion for v1.4 (v2)

This is a follow-up review after the v1 findings were addressed. The fix commit (8c57d2f) resolved 7 of the 8 original issues: `compare_players` now uses batched SQL, `UndoManager` gained a `clear()` method, `_write_fallback_pdf` was promoted to a public API, analytics UI handlers log exceptions, profile deletion logs before acting, the redundant `detect_format` dispatch in drag-drop was removed, and the guided tour per-profile behavior is documented as intentional. The PDF latin-1 limitation is now documented inline.

**Watch for:** The Ctrl+S backup shortcut imports a function (`create_backup`) that does not exist in `backup_restore.py` â€” the resulting `ImportError` is silently caught, making the shortcut completely dead (confirmed). The `UndoManager.clear()` method exists but is never called anywhere in production code, so the closure-retention concern remains unresolved at runtime (confirmed). The `ShortcutManager._trigger_backup` swallows all exceptions with a bare `except Exception: pass`, giving no user feedback on Ctrl+S (confirmed).

**Verdict**: NEEDS_CHANGES

## High-level view

The `UndoManager.clear()` method was added per the v1 request, but no call site invokes it. On profile switch, `set_last_used_profile` updates the registry and shows a "restart required" message â€” the app never switches in-process, so the concern is partially academic for the current UX. If hot-switching is ever added, stale closures holding old DB connections would execute against a closed connection.

The Ctrl+S shortcut is broken at import time: `backup_restore.py` exports `export_profile_backup` (which requires a connection and destination path), not `create_backup`. The bare except swallows the `ImportError`, so pressing Ctrl+S does nothing silently.

## Ctrl+S backup shortcut: broken import

```python
def _trigger_backup(self) -> None:
    from app.services.backup_restore import create_backup

    try:
        create_backup()
    except Exception:  # noqa: BLE001
        pass
```

`backup_restore.py` defines `export_profile_backup(*, connection, destination_path)` â€” a function requiring a DB connection and a file path. There is no `create_backup` symbol. The deferred import means this doesn't fail at startup (it raises `ImportError` at call time), and the bare except hides it completely. The shortcut was never functional.

A minimal fix: create a convenience `create_backup()` in `backup_restore.py` that gets the connection from `get_connection()`, generates a timestamped path under `restore_points_dir`, and delegates to `export_profile_backup`. Then change `_trigger_backup` to show a toast on success/failure instead of silently passing.

## UndoManager.clear() lifecycle gap

Neither `MainWindow`, `ProfileSelectorDialog`, nor any shutdown path calls `undo_manager.clear()`. The current profile switch tells the user to restart, so the undo stack dies with the process. But `MainWindow.closeEvent` (or a shutdown hook) should still call `undo_manager.clear()` to explicitly release captured references â€” closures may hold open file handles or large in-memory structures that benefit from deterministic cleanup rather than waiting for GC.
