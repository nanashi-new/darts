## Backup shortcut connection lifecycle

```python
def create_quick_backup() -> Path:
    from app.db.database import get_connection

    connection = get_connection()
    paths = get_runtime_paths()
    ...
    backup_conn = sqlite3.connect(str(destination))
    try:
        connection.backup(backup_conn)
    finally:
        backup_conn.close()
    return destination
```

The source `connection` is obtained from `get_connection()` and never closed. Elsewhere in the codebase (`ImportExportView.__init__`, `MainWindow._refresh_status_bar`) the same function is called and the result stored without closing—indicating it's a singleton accessor. If it ever creates a fresh handle per call, the backup shortcut would leak one connection each time it fires.

## Help search section splitting

```python
for line in lines:
    if line.startswith("## "):
        if current_header or current_lines:
            sections.append((current_header, current_lines))
        current_header = line
        current_lines = []
    elif line.startswith("# ") and not current_header:
        current_header = line
        current_lines = []
    else:
        current_lines.append(line)
```

The `# Руководство пользователя` title and the blank lines before the first `## Начало работы` get bundled into the first section tuple. When the user searches for "руководство", the match fires on the title, and the returned content includes the first `## ` section body. A search for the guide title shouldn't surface unrelated section content.
