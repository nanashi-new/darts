# P2 Архитектурные Решения

## QTabWidget для карточки игрока
**Решение:** Заменить единый QScrollArea на QTabWidget с 6 вкладками.  
**Причина:** Карточка содержит слишком много информации для одного скролла. Вкладки дают логическую группировку и ускоряют навигацию. Каждая вкладка имеет свой QScrollArea для длинного контента.

## QSS для системы тем
**Решение:** Использовать QSS (Qt Style Sheets) вместо программной настройки палитры QPalette.  
**Причина:** QSS проще поддерживать и расширять. Стили применяются ко всему приложению одной командой. Легко добавить новые темы или настроить цвета.

## Polymorphic entity_tags
**Решение:** Таблица entity_tags с полями (tag_id, entity_type, entity_id) вместо отдельных таблиц player_tags, tournament_tags.  
**Причина:** Единая точка управления тегами. Один сервис обслуживает любые сущности. Меньше кода, проще расширять на новые типы.

## Сохранение backward-compatible атрибутов
**Решение:** При рефакторинге PlayerCardDialog сохранить все публичные атрибуты (overview_label, notes_table, tournament_history_table и т.д.).  
**Причина:** Существующие тесты проверяют эти атрибуты напрямую. Ломать тесты при рефакторинге UI недопустимо.

## QSplitter для списка турниров
**Решение:** Добавить левую панель со списком турниров через QSplitter.  
**Причина:** Пользователь может быстро переключаться между турнирами без перезагрузки всей страницы. Фильтры применяются к списку, а не к единичному турниру.
