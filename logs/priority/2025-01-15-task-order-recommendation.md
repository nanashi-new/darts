# Рекомендация по порядку задач P1-16 и P1-17

**Дата:** 2025-01-15

## Рекомендация

Выполнять P1-16 (безопасность данных, backup/restore) **перед** P1-17 (автоматические переходы между сезонами/лигами).

## Обоснование

1. **Безопасность данных - предпосылка для рискованных операций.**
   Backup/restore (P1-16) является необходимым условием для любой автоматизированной операции, которая массово изменяет данные. Без надежного механизма восстановления любая ошибка в автоматизации может привести к необратимой потере данных.

2. **Сезонные переходы модифицируют назначения игроков в лигах массово.**
   P1-17 реализует автоматические переходы между лигами на основе рейтинга - это bulk-операция, которая изменяет `league_id` у множества игроков одновременно. Механизм backup/restore должен быть отлажен до того, как такие операции станут доступны пользователю.

3. **P1-16 добавляет подтверждения, которые защищают от случайной потери данных.**
   Диалоги подтверждения и точки восстановления, реализуемые в P1-16, будут использоваться действием "Применить" в P1-17. Без них пользователь может случайно запустить массовое переназначение без возможности отката.

## Вывод

Порядок: P1-16 -> P1-17. Это обеспечивает наличие "страховочной сетки" перед реализацией потенциально деструктивных операций.
