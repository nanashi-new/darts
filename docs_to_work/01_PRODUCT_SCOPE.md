# Product Scope
## Система рейтинга дартс Тверской области

**Редакция пакета:** v3  
**Связанный индекс:** `00_INDEX.md`


## 1. Назначение документа

Документ фиксирует рамки продукта:
- что именно мы строим;
- в каком формате поставляется система;
- что входит в finished-версию;
- что не должно неожиданно появиться в архитектуре.

## 2. Краткое определение продукта

Продукт — единое локальное desktop-приложение под Windows для:
- ведения турниров;
- расчёта детских и взрослых рейтингов;
- сопровождения категорий и лиг;
- хранения истории игроков и рейтингов;
- ведения notes / coach context;
- экспорта и печати;
- диагностики и восстановления.

## 3. Главная идея продукта

Это не просто калькулятор рейтинга, а рабочая система сопровождения турниров, игроков, тренеров и административных действий.

## 4. Формат продукта

### 4.1. Что это за продукт
- desktop-приложение;
- offline-first;
- Windows-first;
- локальная эксплуатация;
- без обязательной внешней инфраструктуры.

### 4.2. Что это не за продукт
- не web-приложение;
- не клиент к обязательному удалённому API;
- не оболочка над PostgreSQL;
- не многосервисная инфраструктура.

## 5. Жёсткие рамки

Обязательно фиксируется:
- Windows runtime;
- SQLite как локальная БД;
- один основной exe как целевой релизный артефакт;
- пользовательские данные вне exe;
- offline build при подготовленном wheel-cache.

Запрещено делать обязательными:
- PostgreSQL;
- pgAdmin;
- Docker;
- backend;
- nginx;
- cloud-инфраструктуру.

## 6. Предметная область

Система должна поддерживать:
- детские рейтинги по возрасту и полу;
- взрослый общий рейтинг;
- взрослый мужской рейтинг;
- взрослый женский рейтинг;
- лиги Премьер / Первая;
- очки по классификации и местам;
- rolling rating по последним N турнирам;
- возрастные переходы;
- историю игрока и рейтинга;
- импорт и ручной режим;
- аудит и диагностику.

## 7. Бизнес-задачи

Система обязана решать:
- создание, импорт, проверку и публикацию турниров;
- расчёт рейтингов;
- ведение истории;
- explainable corrections;
- экспорт;
- сопровождение игроков;
- служебный контекст;
- восстановление и release discipline.

## 8. Основные роли

Логически система поддерживает:
- администратора;
- оператора импорта;
- тренера;
- судью / организатора;
- пользователя просмотра рейтингов.

## 9. Что входит в finished-версию

### Доменные блоки
- категории;
- возрастные переходы;
- детская классификация;
- place points;
- rolling rating;
- взрослые рейтинги;
- лиги;
- snapshots и history.

### Рабочие блоки
- турниры;
- импорт;
- ручной режим взрослых;
- player card;
- notes / coach context;
- export / print;
- audit / recovery / diagnostics.

### Эксплуатационные блоки
- offline build;
- one-file exe;
- bat-скрипты;
- restore points;
- self-check;
- startup diagnostics;
- release bundle;
- документация.

## 10. Что не должно входить в первый finished-релиз

Допустимо позже, но не должно размывать scope:
- installer как обязательный слой;
- auto-update;
- cloud sync;
- многопользовательская удалённая работа;
- сложная аналитика вне core-задач.

## 11. Runtime и build stack

Целевой стек:
- Python
- PySide6
- SQLite
- локальная файловая система
- PyInstaller

Build stack:
- pinned requirements
- wheel-cache
- manifest/hash validation
- bat-driven build flow

## 12. Ограничения по данным

Вне exe:
- БД;
- настройки;
- заметки;
- exports;
- logs;
- attachments;
- branding files.

Внутри приложения:
- иконки;
- встроенные шаблоны;
- fallback FAQ;
- bundled defaults.

## 13. UX-рамки

Finished-продукт должен быть:
- понятным;
- объяснимым;
- устойчивым;
- без тупиковых экранов;
- с dashboard;
- с настройкой рабочего пространства;
- с diagnostics/recovery, доступными пользователю.

## 14. Качество и сопровождение

Продукт должен быть:
- документированным;
- тестируемым;
- воспроизводимо собираемым;
- сопровождаемым;
- пригодным к clean-machine release.

## 15. Итоговая фиксация

Финально принимается:
- продукт = единое desktop-приложение под Windows;
- продукт = offline-first;
- внешняя инфраструктура не обязательна;
- SQLite допустима и целевая;
- пользователь получает finished-продукт в форме удобного exe;
- notes, diagnostics, recovery и build входят в finished-версию.
