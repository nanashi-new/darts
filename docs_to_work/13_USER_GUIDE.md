# User Guide
## Руководство пользователя

**Редакция пакета:** v3  
**Связанный индекс:** `00_INDEX.md`


## 1. Для кого документ

Для:
- администратора;
- оператора импорта;
- тренера;
- судьи / организатора;
- пользователя просмотра рейтингов.

## 2. Что это за приложение

Это локальное Windows desktop-приложение для:
- турниров;
- рейтингов;
- игроков;
- истории;
- notes / coach context;
- export/print;
- diagnostics/recovery.

## 3. Первый запуск

При первом запуске приложение:
- создаёт рабочий профиль;
- создаёт настройки;
- создаёт локальную БД;
- создаёт рабочие папки;
- открывает главное окно.

После первого запуска стоит:
- открыть настройки;
- проверить путь к нормативам, если используется;
- выбрать первый рабочий сценарий: турнир или import.

## 4. Главные разделы

- Главная / Dashboard
- Турниры
- Импорт
- Детские рейтинги
- Взрослые рейтинги
- Лиги
- Игроки
- История / аналитика
- Заметки / контекст
- Отчёты / экспорт
- Журнал / аудит
- Диагностика / восстановление
- Настройки
- О программе / справка

## 5. Типовой рабочий сценарий

1. создать или открыть турнир;
2. импортировать данные или ввести их вручную;
3. проверить warnings/errors;
4. пересчитать;
5. проверить влияние на рейтинг;
6. подтвердить и опубликовать;
7. открыть рейтинг / карточку игрока;
8. сделать export;
9. при необходимости оставить note.

## 6. Турниры

### Что такое турнир
Центральная рабочая сущность, вокруг которой строятся:
- import;
- manual input;
- recalc;
- publish;
- history;
- notes;
- audit;
- export.

### Какие турниры бывают
- детский из протокола;
- взрослый manual;
- лиговый;
- черновой;
- по шаблону;
- клон.

### Статусы
- Draft
- Review
- Confirmed
- Published
- Archived
- Cancelled
- Deleted

### Что можно делать с турниром
- редактировать;
- пересчитать;
- подтвердить;
- опубликовать;
- клонировать;
- архивировать;
- удалить;
- экспортировать.

## 7. Импорт

Типовой flow:
1. выбрать файл/файлы;
2. проанализировать структуру;
3. применить import profile;
4. проверить mapping;
5. пройти player matching;
6. посмотреть preview;
7. прочитать warnings/errors;
8. сохранить в draft или применить.

Import profile помогает повторно использовать mapping для похожих файлов.

Если matching неоднозначен, можно:
- выбрать существующего игрока;
- создать нового;
- сохранить правило выбора.

## 8. Взрослый ручной режим

Используется, если турнир вводится вручную.

Flow:
1. создать взрослый турнир;
2. добавить игроков;
3. ввести очки и места;
4. проверить totals;
5. опубликовать;
6. пересчитать взрослый рейтинг.

## 9. Рейтинги

Система поддерживает:
- детские рейтинги;
- взрослый общий;
- взрослый мужской;
- взрослый женский;
- лиги.

Пользователь может:
- выбрать scope;
- выбрать N;
- искать игрока;
- открыть player card;
- export/print.

## 10. Игроки

Пользователь может:
- искать;
- создавать;
- редактировать;
- удалять/архивировать;
- открывать историю;
- использовать merge duplicates.

Finished player card должна показывать:
- историю турниров;
- историю рейтинга;
- category transitions;
- league history;
- notes;
- coach context;
- training journal.

## 11. Заметки и context

Заметки нужны для:
- служебных комментариев;
- спорных кейсов;
- follow-up;
- coach observations.

Coach mode нужен для:
- coach notes;
- training journal;
- целей;
- progress tracking.

## 12. Export и печать

Можно экспортировать:
- рейтинг;
- турнирный протокол;
- grouped exports;
- batch exports;
- внутренние отчёты.

Основные форматы:
- PDF
- XLSX

## 13. Журнал и аудит

Журнал — formal trace системы:
- import;
- recalc;
- export;
- merge;
- errors.

Это не то же самое, что notes.

## 14. Diagnostics и recovery

Пользователь должен уметь:
- открыть журнал;
- открыть diagnostics;
- собрать diagnostic bundle;
- посмотреть restore points;
- выполнить safe reset profile, если это действительно нужно.

## 15. Типовые сценарии

### Детский турнир
- create tournament
- import
- check matching
- preview
- recalc
- publish
- export

### Взрослый турнир
- create adult tournament
- enter results
- recalc
- publish

### Проверка игрока
- find player
- open card
- review history
- add note if needed

### Исправление ошибки
- locate relevant entity
- use correction flow
- add reason
- inspect affected changes
- use recovery/diagnostics if needed

## 16. Что делать, если что-то пошло не так

- не удалять профиль сразу;
- сначала открыть журнал;
- понять последнюю проблемную операцию;
- затем решать, нужен ли correction, restore или reset.

## 17. Рекомендации пользователю

- работайте сценариями, а не хаотичными кликами;
- читайте warnings;
- не удаляйте данные без необходимости;
- используйте журнал и diagnostics при спорных кейсах.

## 18. Итог

User guide finished, если:
- по нему новичок может начать работу;
- по нему можно пройти ключевые сценарии;
- он не требует чтения исходного кода;
- он покрывает tournaments, import, ratings, players, notes, export и recovery.
