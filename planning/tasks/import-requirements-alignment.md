# Требования импорта: XLSX, aliases, ошибки и review-flow

Статус: done  
Приоритет: P0  
Источник: `planning/reference/customer-requirements/2026-02-02-rating-system-requirements-approved.md`

## Цель

Сверить и закрыть требования заказчика по импорту турнирных протоколов из Excel.

## Что меняем

- Проверить обязательные колонки: место, ФИО, дата рождения или год рождения, набор очков, сектор 20, большой раунд.
- Проверить aliases и регистр названий колонок.
- Проверить отчет об ошибках импорта и понятный review-flow перед публикацией.
- Проверить сценарий нескольких таблиц в одном XLSX.
- Проверить ручной выбор игрока при неоднозначном совпадении.

## Что не трогаем

- Не добавляем новые внешние форматы импорта в P0.
- Не меняем структуру импортируемых Excel-шаблонов без обратной совместимости.
- Не публикуем импорт автоматически без review-flow.

## Тесты и проверки

- Targeted tests для `app/services/import_xlsx.py`.
- Targeted tests для import review/apply dialogs.
- Ручной smoke: один корректный XLSX, один XLSX с ошибками, один XLSX с нестандартными aliases.
- `pytest -q` после реализации.

## Критерии готовности

- Обязательные колонки и aliases покрыты тестами.
- Ошибки импорта видны пользователю и сохраняются в отчетах.
- Импорт создает черновик/предпросмотр, а публикация остается явным действием.
