# Task: Data Safety And Backups

Статус: done

## Цель

Сделать сохранность данных понятной и проверяемой для пользователя.

## Что Меняем

- Улучшить видимость состояния профиля, restore points и диагностики.
- Сделать понятные сообщения перед рискованными действиями.
- Подготовить release/manual checks для backup/restore сценариев.

## Что Не Трогаем

- Не менять формат БД без отдельной миграционной задачи.
- Не добавлять облачную синхронизацию.
- Не удалять существующие restore-сценарии.

## Тесты И Ручные Проверки

- Smoke диагностики профиля.
- Проверка создания restore point.
- Проверка clean-profile и восстановления на тестовом профиле.

## Критерии Готовности

- Пользователь понимает, где его данные и как их защитить.
- Перед опасными действиями есть русские подтверждения.
- Release checklist покрывает backup/restore.
