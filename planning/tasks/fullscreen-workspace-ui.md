# Task: Fullscreen Workspace UI

Статус: done

## Цель

Сделать «Дартс Лига» приложением, которое по умолчанию открывается как полноценное рабочее desktop-окно и использует больше места для таблиц, карточек, диагностики, импорта и отчетов.

## Что Меняем

- При первом запуске открывать главное окно в максимизированном режиме (`showMaximized`), а не маленьким окном 1024x640.
- Не делать borderless/fullscreen game-mode: оставить рамку окна, Alt+Tab, панель задач и стандартные кнопки Windows.
- Сохранять выбранную вкладку как сейчас, но не восстанавливать маленький размер как дефолт нового профиля.
- Пересмотреть главные вкладки под широкий экран:
  - `Главная`: компактный dashboard с быстрыми действиями и статусами.
  - `Рейтинг`: таблица занимает основное место, фильтры и действия не раздувают высоту.
  - `Турниры`: список, детали и результаты должны лучше использовать ширину.
  - `Игроки`: список и карточка/действия должны быть удобны на широком экране.
  - `Импорт/Экспорт`, `Диагностика`, `Настройки`: длинные блоки получают скролл или разделение на панели.
- Добавить понятные минимальные размеры и stretch/sizePolicy там, где сейчас русские строки могут сжимать таблицы или кнопки.

## Что Не Трогаем

- Бизнес-логику рейтинга, импорта и экспорта.
- Формат БД и пользовательские данные.
- Новые P2 функции: теги, вложения, кастомные поля.
- Глубокий редизайн бренда, цвета и иконок.

## Правила Реализации

- Первый слой задачи - запуск через `showMaximized`, минимальный размер и базовые sizePolicy для главного окна.
- Второй слой - точечная оптимизация вкладок, где широкий экран реально помогает: таблицы, списки, импорт, диагностика, настройки.
- Не превращать задачу в полный redesign: если нужно менять визуальный стиль, это отдельная задача после v1.1.
- Не добавлять новые пользовательские сценарии. Разрешены только изменения компоновки, доступности текста и удобства существующих действий.
- Если не помещается русский текст, сначала сокращать подпись и добавлять tooltip; затем добавлять scroll/split/modal, если проблема остается.

## Тесты И Ручные Проверки

- UI smoke: главное окно создается, вкладки на русском, нет placeholder-текста.
- Новый тест или расширение smoke: для нового профиля окно стартует в maximized/fullscreen-like рабочем состоянии.
- Ручная проверка на 1920x1080 и 1366x768:
  - вкладки не обрезаются;
  - основные таблицы получают максимум полезного места;
  - кнопки не распирают панели;
  - при уменьшении окна появляется скролл или корректная адаптация.
- `git diff --check`.

## Критерии Готовности

- Приложение по умолчанию открывается максимизированным.
- Основные экраны выглядят рассчитанными на широкий рабочий режим, а не как растянутое маленькое окно.
- Пользователь может уменьшить окно без визуального развала интерфейса.
- UI polish task можно выполнять точечно поверх этой основы.
