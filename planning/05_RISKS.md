# 05 - Риски

Этот файл помогает видеть, что может сорвать релиз или ухудшить качество продукта.

## P0 Риски До v1.1

| Риск | Влияние | Снижение |
|---|---|---|
| Локальное Python/Qt окружение может быть не подготовлено на новой машине | Нельзя подтвердить тесты перед PR | Использовать `.local/python312` + `.venv` или повторить setup; runtime-папки не коммитить |
| Maximized UI может скрыть тесные места на малых окнах | Пользователь уменьшит окно и увидит развал интерфейса | Проверять 1920x1080, 1366x768 и уменьшенное окно |
| Русские подписи длиннее английских | Кнопки и таблицы могут распирать панели | Короткие команды, tooltip, scroll/split/modal |
| Старые термины могут остаться в активных файлах | Бренд и смысл продукта будут выглядеть нечищеными | Проверять `rg` по активным product-файлам без `planning/archive/` |
| Подтвержденный документ заказчика содержит исторический бренд и старую классификацию | Можно случайно вернуть старый бренд или старые нормативы при сверке требований | Использовать `planning/04_DECISIONS.md` как приоритет над superseded-частями документа; сверку вести через `customer-requirements-alignment` |
| Реализация может расходиться с обязательными сценариями из документа заказчика | Release будет удобным, но не полностью соответствующим согласованным требованиям | Перед `release-readiness` пройти reference-файл, проверить rating/import/export/tournament/player/audit flow и зафиксировать расхождения |
| Installer зависит от Inno Setup | Нельзя собрать основной артефакт на любой машине | Оставить ZIP/exe fallback и явно писать зависимость в release checklist |

## P1/P2 Риски

| Риск | Влияние | Снижение |
|---|---|---|
| Новые функции начнутся до закрытия v1.1 | Релиз расползется | P2 не начинать до release-ready |
| Карточка игрока станет слишком большой | Экран будет тяжелым и непонятным | Делить на вкладки/секции, держать основную карточку сканируемой |
| Вложения и кастомные поля усложнят данные | Риск миграций и поломки профилей | Проектировать отдельной задачей с миграцией и backup-check |
| Тренерские сценарии смешаются с турнирами | Размытая навигация | Отделить coach workflow от tournament workflow |

## Правило Обновления

- После каждого найденного риска добавить строку сюда.
- После закрытия риска перенести его в отдельный раздел `Закрытые Риски` или удалить только после релиза.
