# 00 - Индекс Документации

Активная разработка начинается с [`planning/00_PRIORITY.md`](planning/00_PRIORITY.md).

## Активные Документы

1. [`planning/00_PRIORITY.md`](planning/00_PRIORITY.md) - порядок выполнения задач.
2. [`planning/01_RULES.md`](planning/01_RULES.md) - правила работы, качества, тестов и релиза.
3. [`planning/02_ROADMAP.md`](planning/02_ROADMAP.md) - развитие по версиям.
4. [`planning/03_CURRENT_STATE.md`](planning/03_CURRENT_STATE.md) - текущее состояние ветки.
5. [`planning/tasks/`](planning/tasks/) - конкретные task-файлы.
6. [`10_RELEASE_CHECKLIST.md`](10_RELEASE_CHECKLIST.md) - релизный gate, не очередь разработки.

## Архив

Старые планы и исторические отчеты перенесены в [`planning/archive/`](planning/archive/):

- `root-docs/` - прежние корневые документы `01_*`-`09_*` и дизайн-обзор.
- `docs-to-work/` - старые master/user/admin/release материалы.
- `release-artifacts/` - исторические release run, smoke и manual отчеты.

Если информация из архива нужна для новой работы, сначала перенести ее в `planning/00_PRIORITY.md` или отдельный task-файл.
