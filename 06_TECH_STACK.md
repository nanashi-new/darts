## 1. Основные принципы выбора стека

- Максимально **подходит для работы с Codex**
- Прост в поддержке одним разработчиком
- Хорош для Excel, PDF и UI
- Не требует серверов и сложной инфраструктуры
- Легко собирается в `.exe`

---

## 2. Язык программирования

### Python 3.12+

Выбран потому что:
- Codex лучше всего генерирует Python-код;
- отличная работа с Excel, SQLite, PDF;
- огромная экосистема библиотек;
- быстрый прототипинг и доработка.

---

## 3. GUI (интерфейс)

### PySide6 (Qt 6)

Причины:
- современный внешний вид;
- высокая производительность;
- полноценные таблицы (QTableView);
- печать и экспорт в PDF из коробки;
- отлично подходит для десктопа Windows.

---

## 4. База данных

### SQLite (встроенная)

- файл внутри приложения (`app.db`);
- не требует установки;
- надёжна и быстра;
- легко переносится между ПК.

### Доступ к БД
- стандартный `sqlite3`  
или  
- `SQLAlchemy` (рекомендуется для чистой архитектуры).

---

## 5. Импорт / экспорт Excel

### openpyxl

Используется для:
- чтения `.xlsx`;
- записи `.xlsx`;
- сохранения форматирования;
- автоширины колонок.

Дополнительно (опционально):
- `pandas` — для ускорения batch-импорта.

---

## 6. PDF, печать, изображения

### Печать и PDF
- `Qt PrintSupport`
- вывод таблиц напрямую в PDF
- WYSIWYG (что видишь — то печатается)

### Экспорт как изображение
- рендер таблицы в `QPixmap`
- сохранение в PNG/JPG
- удобно для отправки в мессенджеры

---

## 7. Упаковка приложения

### PyInstaller

- сборка в один `.exe`;
- без консоли;
- работает без установленного Python;
- подходит для передачи судьям.

## 8. Структура проекта (рекомендуемая)
